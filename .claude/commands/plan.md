# /plan

Enter planning mode for complex tasks.

## When to Use
- Task has multiple steps
- Uncertainty about approach
- Previous attempt failed
- Task touches multiple files/systems

## Planning Template

```markdown
## Task: [Brief description]

### Goal
What does success look like?

### Constraints
- Time: 
- Dependencies:
- Must not break:

### Approach Options
1. **Option A:** [description]
   - Pros: 
   - Cons:
   
2. **Option B:** [description]
   - Pros:
   - Cons:

### Chosen Approach
[Option X] because [reason]

### Steps
1. [ ] Step 1
2. [ ] Step 2
3. [ ] Step 3

### Verification
How will we know it works?
- [ ] Test 1
- [ ] Test 2

### Rollback Plan
If things go wrong:
1. [recovery step]
```

## After Planning
- Save plan to WORKING.md
- Get approval if needed
- Execute step by step
- Check off as you go
