## 📝 KEY FINDINGS

1. **Galxe uses lazy authentication:**
   - Initial connection: Just checks `eth_accounts` (no signature)
   - Signature requested only when: claiming quests, binding accounts, etc.

2. **Provider injection timing:**
   - Must inject BEFORE page load (`evaluateOnNewDocument`)
   - EIP-6963 announcement is critical for modern wallet detection

3. **Signature queue system:**
   - Window.__walletSignatureQueue stores pending requests
   - Node.js polls and signs, then resolves the promise
