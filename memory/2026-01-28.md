### Architecture
```
Layer 1: Knowledge Graph   (/life/areas/)
  └── Entities with atomic facts + living summaries

Layer 2: Daily Notes       (memory/YYYY-MM-DD.md)
  └── Raw event logs — what happened, when (already existed)

Layer 3: Tacit Knowledge   (MEMORY.md)
  └── Patterns, preferences, lessons learned (already existed)
```

### Updated Files
- `AGENTS.md` — Added Three-Layer Memory System documentation
- `HEARTBEAT.md` — Added fact extraction on heartbeats
- `memory/heartbeat-state.json` — Added lastFactExtraction tracking

### How It Works
- **Atomic facts** stored in `items.json` with timestamps and status
- **Living summaries** rewritten weekly from active facts
- **Superseding** instead of deleting — full history preserved
- **Tiered retrieval** — load summary.md for quick context, items.json for full history

### Why This Matters
- Context stays current without manual edits
- Facts are structured and searchable
- History is preserved (superseded, not deleted)
- Weekly synthesis keeps summaries lean
