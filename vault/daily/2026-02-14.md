# 2026-02-14 — Server Recovery & Hardening

## Incident: DatevBereit Down (~24h)

### Timeline
- **Feb 12, ~22:08 UTC** — Docker stack went down during CC work. Rebuilt containers.
- **Feb 13, 00:30-07:28 UTC** — Site working. Arber used dashboard at 07:28.
- **Feb 13, 07:28 → Feb 14, 07:10 UTC** — Web container silently became unresponsive.
- **Feb 14, 07:10 UTC** — Max investigated, diagnosed, fixed.

### Root Causes (3 stacked)
1. **Next.js running in dev mode** (`pnpm dev` via `Dockerfile.dev`) — fragile under load
2. **Docker bridge networking broken by Tailscale** — DNS failed inside containers, hot-reloader broke
3. **`qmd embed` processes eating 400% CPU** — from OpenClaw update re-indexing embeddings, starved Next.js

### Additional Issue
- **Tailscale exit node was active** — routed all traffic through Arber's Windows PC, broke inbound connections from outside

## Fixes Applied
- ✅ Web container switched to `network_mode: host` (bypasses Docker networking)
- ✅ DNS fix: added `registry.npmjs.org` to container hosts
- ✅ Killed `qmd embed` processes, created `nice -n 19` wrapper
- ✅ Production Dockerfile built (`Dockerfile.prod` — multi-stage, `next build` + `next start`)
- ✅ Tailscale exit node removed permanently
- ✅ Supermemory extension removed (was causing issues since OpenClaw update)
- ✅ CUPS disabled (unnecessary print server on VPS)
- ✅ Postfix locked to localhost only (was open on 0.0.0.0:25)

## New Monitoring
- **datev-healthcheck** — every 5 min liveness ping, alerts on failure
- **daily-security-check** — 8:00 AM CET, full audit (ports, SSH, Docker, disk, CPU, SSL, Tailscale)

## Security Audit Result
- Ports 80/443: open (nginx) ✅
- Port 22: SSH locked to home IP + Tailscale ✅
- Firewall: default DROP policy ✅
- Docker ports (8000, 5432, 6379): localhost only ✅
- No exit node active ✅
- OpenClaw v2026.2.13 (latest beta) ✅

## Decisions
- **Tailscale exit node removed permanently** — site uptime > Reddit proxy
- **Supermemory removed** — was competing with memory-core, causing silent failures
- **Production Dockerfile created** — needs to be deployed (currently still dev mode but stable)
