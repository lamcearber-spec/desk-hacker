### Fixes
1. **API 502 (Bad Gateway)** — nginx was proxying `/api/` to port 8001 but API container runs on 8000
   - Fix: `sed -i 's/proxy_pass http:\/\/127.0.0.1:8001/proxy_pass http:\/\/127.0.0.1:8000/' /etc/nginx/sites-available/datev-bereit.conf`

2. **Celery Redis Connection Error** — "Retry limit exceeded while trying to reconnect"
   - Root cause: Celery config builds URLs using `REDIS_HOST` which defaulted to `localhost`
   - Container had `REDIS_URL=redis://redis:6379/0` but Celery properties used `REDIS_HOST` directly
   - Fix: Added `REDIS_HOST=redis` to docker-compose.yml environment for api/worker/beat services
   - Also updated `.env` file: `REDIS_HOST=redis` and `REDIS_URL=redis://redis:6379/0`

### Key Learning
Docker networking: Services must use container names (e.g., `redis`) not `localhost` when communicating between containers. The Celery config's `celery_broker_url` and `celery_result_backend` properties build URLs from `REDIS_HOST`, not `REDIS_URL`.
