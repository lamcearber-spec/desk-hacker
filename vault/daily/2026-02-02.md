## 🚨 Incident: Tool-ID Desync / Poisoned Session (10:45 UTC)

**Symptoms:**
- Error: `unexpected tool_use_id found in tool_result blocks`
- Every message triggered API rejection
- Server I/O lock, terminal freeze

**Root Cause:**
- Sync error between Anthropic API and local session history
- A tool result was sent with an ID the API didn't recognize
- Corrupted conversation history kept reloading on every message

**Fix (Rescue Mode Required):**
1. Boot server into Hetzner Rescue Mode (RAM-based)
2. Mount disk: `mount /dev/sda1 /mnt`
3. Delete sessions: `rm -rf /mnt/root/.clawdbot/agents/main/sessions/`
4. Reboot to normal mode

**Quick Fix (if disk not locked):**
```bash
systemctl stop clawdbot-gateway  # or kill the process
rm -rf ~/.clawdbot/agents/main/sessions/
# or just: rm ~/.clawdbot/agents/main/sessions.json
systemctl start clawdbot-gateway
```

**Prevention:**
- Monitor for tool_use_id errors in logs
- If API starts rejecting all messages, check session state immediately
